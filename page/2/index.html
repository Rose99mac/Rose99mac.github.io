值得学习的工作守则

发表于 2016-02-16

如何干好工作

  1. 无论最初的工作任务多么卑微、琐碎，都应竭尽所能、不遗余力地去完成

  2. 任何时候都能把领导交办的任务做好

  3. 主动跟进工作，督促合作方工作进展

  4. 明确职责，必要时以书面形式加以确认提醒

  5. 当收到不满意反馈或另有其他要求时，应坚持到底、尽善尽美地完成工作

  6. 要有自己的立场，避免犹豫不决、摇摆不定

  7. 不要胆怯，要大胆地表现自己，提出自己的想法

  8. 请上司批准你的行动之前，先制定好翔实的计划与可行的方案

  9. 口头报告与书面报告都应力求简洁、明确

10.一定要注意表述内容的正确性

如何处理好与上司的关系

11.每位员工都应了解自己职权范围内工作的进展情况

12.不要忘了你是在为上司工作

13.选择上司要谨慎，自己不能选择时就选择适应

14.及时向上司报告自己工作的重大进展

15.将上司交办的事放在第一位

16.不要盲目遵从上司的指令，不要总是上司说什么才做什么

如何处理好与同事及客户的关系

17.未经主管人员许可，不要干涉其他部门和人员的工作

18.工作中，要“谨慎”对待与自己工作相关的每一个人

19.不过问他人的隐私，不传播令他人难堪的“消息”

20.工作中，做出承诺、保证进度和预测未来都是十分重要的职责

21.当你对其他部门或他人的工作不满意时，应当直接向当事人进行投诉

22.与客户及外部人员打交道时，应记住你代表着公司

如何成为工作中最受欢迎的人

23.能够与各类性格的人友好相处

24.养成一有机会就帮助其他同事的习惯

25.在任何情况下都应力求公平公正

26.对待工作不要过于一本正经

27.大方一点，见到他人要热情

28.当你对他人的动机有所怀疑时，在自己能控制的情况下，尽量相信他们

29.工作中不必过于和蔼可亲

30.将个人诚信视作自己最宝贵的资产之一

31.一丝不诚信也会带来大麻烦

32.注重自己的个人形象非常重要

33.工作中也需要做到知己知彼

如何当好主管

34.学会授权，不要事必躬亲

35.工作时要分清主次，先完成重要之事

36.对于工作，要有化繁为简的习惯

37.慎重对待“紧急”工作，对于紧急工作要考虑清楚再接手

38.做好会议准备，让每次会议都有效

39.养成提纲挈领、简化会议的习惯

40.在宣布重大决议或决策前，要做好必要的准备

41.事先要制定计划，然后按计划去执行

42.即使在工作进展顺利时，也不要轻易否定新提议和新方案

43.要经常回顾工作的进展情况，确保每项工作的收益与成本都成正比

44.工作需要定期总结，必要时需要提交总结报告

45.不要让很多人直接向某一个人汇报工作

46.职权划分要明确，工作分工要合理

47.如果自己的法定权力不够，应尽量争取所需的权力

48.不要固步自封，而应保持进取心

49.尊重下属就是尊重自己

管理者应给予下属的东西

50.经常鼓励下属的个人兴趣和专业兴趣

51.不要自私地限制下属的发展和前途

52.尽量避免打断别人的谈话或无视他人的存在

53.要及时通知下属他们应当知晓的信息

54.注意自己的工作方式和方法，不要在他人面前批评下属

55.经常对下属正在进行的工作表示关怀

56.下属出色完成任务后，要给出你的赞赏或奖励

57.必要时，要勇于替自己的团队及成员承担相应的责任

58.了解和掌握下属对工资及其他待遇的满意程度

59.举办活动时，要确保每个客人都能尽兴

60.尽可能维护自己下属及其家庭的利益，帮助他们解决实际困难

学习App设计总结

发表于 2016-02-01

不要过度教育你的用户 (引导页不需要)
用户下载App以后期望马上可以使用它 )
处理方法 : 把引导页作为UI的使用指导

简化页面内容,去除与核心内容不相关信息

保证每个界面的信息都是清楚且唯一的,若一个界面有超过一种操作,区分优先级,避免让用户以为两个按钮在同一个等级中,以避免误操作
处理方法: 每个界面的主要操作设计样式保持一致或关联。用户习惯惯性思维，如果你的设计干扰了它的思维方式，那么你很可能失去它们。 当你删除所有不必要的和会分歧的内容遗憾，用户在你的页面体验过程是以秒来计算，而不是以分钟。app首先要满足最核心的需求，用户能得到最高效的体验，之后在考虑添加一些辅助功能和需求。

及时的向用户做出积极的反馈

视觉上和细节上告知用户使用我们app完成某个功能需要多少操作
处理方法:（例如进度蓝条 页码等方式）

找出并关注你的推荐渠道

产品初期，推荐将是你最有力的用户增长来源。为了保持增长，开始计算每个用户的推荐数。然后根据推荐渠道的转化率进行优先级排序
邀请成功数转化率 ＝ ( 点击数／邀请数 ) x（下载数／点击数）

让你的推荐渠道实现双赢

高质量的推荐渠道中推荐者和被推荐者都是app的目标用户(比如都是大学生).一个探索推荐模式的方法是去考虑一下用户在你的app之外,以及围绕着你的app都在干什么.

考虑同时给双方优惠
赠送奖励,推荐者给予奖励,这种推荐方式易于传播也能被大众用户所接受,并且很容易在短时间内吸引大量的目标用户注册,用简单的低成本的方法来测试这类推荐的方法,发布一个推荐奖励的活动,设定活动紧紧维持一个时间段,并设定预期目标.然后看这次活动的实际效果,如何很好,可以把这个活动变成永久性的.(最小化产品)

不要过度依赖社交媒体分享
不要把社交分享放在很重要的地方,即使产品并没有社交属性.它们并没有什么价值,也不会产生什么效果.如果必须添加这个功能,仔细考虑下,并把它和主要功能联系起来.

以稳定增长为目标
找出一到两个主要的优质用户渠道来源,然后让更多的用户,更容易的通过这些渠道入驻app,设计更清晰的漏斗型事件监视数据模型,密切兼职每个页面用户出发的浏览行为.应该从用户第一次下载app开始,一直到他完成产品的核心功能操作.测试用户从第一天到第30天的用户存留情况.在这短时间内有多少比例的新用户依旧留在app里?有些原因或者因素促使你的用户持续使用你的产品?如果有,那么在你的产品里强化它们. 发现最核心的目标用户,它们信仰你的产品.当你找到它们时,接下来要做的就是说服那些非目标用户.

社交类 内容 = 下载量(注册量/下载量)(内容发布者量/注册量)*(内容量/内容发布者量)

电子商务类 购买量 = 下载量 (注册量/下载量) (购买用户数/注册量)*(购买量/购买用户数)

用户下载我们的app是用来使用的 而不是阅读的。

忽略占位图
“当用户没有设置个人头像时他的默认头像是什么样子的？如果他没有任何朋友时他的联系人页是什么样子的？或者他的购物车里没有任何商品时是什么样子？相当一部分app没有对空状态做出设计，导致用户离开。” “这些失误存在的原因是因为app设计者们并没有专门考虑过这些使用情景，事实上一般情况下新用户刚刚使用app时几乎都会遇到这些使用情景。

注册登陆
退出登录时应保留上一次用户的账号，便于后面快速登录。用户花费有限的精力在你的app上，因此尽量减少它们大量去花费思考的精力。确保它们有足够的精力去探索app更加重要、有趣的功能和操作。“如果有两个选项很接近，考虑将其中一种作为主要方式，另一种作为第二选项，但不应该多于两种方式。”

键盘出现时效果
键盘出现时会不会遮盖住一些关键信息或者链接，导致用户需要滑动屏幕才能够看到？是否遮盖住了主要操作按钮？永远不要让这些情况发生！此外，还要考虑切换键盘会带来的加载时间问题。

没有关注错误状态的展示
当用户错误输入信用卡号时会出现什么信息？输入邮箱地址的格式错误时会出现什么信息？在填写表单时会出现各式各样的错误，怎样去提示用户？怎样让用户更简单有效的去修改错误？有时这些提示信息出现的位置和内容令人迷惑，值得去研究并重新设计
